## Project Overview

**Paths Untold** is a Human-in-the-Loop, Stateful LLM Narrative Environment — a choice-driven story game where an LLM generates dynamic scenes, player choices feed back into persistent memory, and characters evolve emotionally over time. It's simultaneously a playable game and a research prototype for studying long-horizon narrative coherence and human-AI interaction.

## Naming Convention

These are the canonical domain terms used throughout the codebase. Use these names in all new code and documentation:

| Term | Meaning | Old name (before rename) |
|------|---------|--------------------------|
| **Run** | A complete game playthrough | — |
| **Chapter** | A major narrative arc (`arc.chapter`) | — |
| **Scene** | One LLM-generated story beat; a node in the narrative graph | Scene / story segment |
| **Beat** | Narrative pacing unit within a chapter (`arc.beat`) | — |
| **Path** | A player choice option presented after a scene | Choice / choice option |
| **Prose** | The story text generated by the LLM for a scene | Story / story text |
| **Memory** | The `GameMemory` object holding all persistent state | GameMemory |
| **Segment** | A rendered unit in the UI display (prose or choice separator) | Story segment / storySegment |
| **Packet** | The normalized JSON object returned by the LLM for one scene | AI response / raw AI |

### Key field names in `GameMemory`
- `prose: string[]` — array of scene prose strings (one per scene in linear path)
- `paths: string[]` — array of player choice strings taken at each scene
- `sceneIndex: number` — index of the current scene
- `summary: string[]` — LLM-generated summaries for memory compression
- `companions: Companion[]` — character relationship state
- `world: WorldState` — location, objectives, scene tags, clock
- `arc: { chapter, beat, tension }` — narrative pacing

### Key field names in narrative graph nodes
- `prose: string` — raw prose text
- `proseHtml: string` — HTML-escaped prose for rendering
- `paths: string[]` — the 4 player choice options for this scene

### Key function names
- `buildScenePrompt()` — builds `{ system, user }` messages for the LLM
- `generateScene()` — non-streaming LLM call
- `generateSceneStream()` — streaming LLM call (infrastructure ready, display not used)
- `handleSceneResponse()` — processes raw LLM output into a new graph node
- `buildSceneSegments()` — builds the rendered `segments[]` array from graph path
- `extractAndNormalizeAiResponse()` — parses LLM JSON into a normalized Packet

## Architecture

### Core Game Loop

```
Player selects a Path
  → State Update (GameMemory in GameScreen.jsx)
  → buildScenePrompt.js (compile world + arc + companions + compressed history)
  → llmClient.js → POST /api/chat → Express proxy → OpenAI API
  → storyParser.js (extract JSON Packet from LLM response)
  → updateFromAIPacket.js (merge Packet into GameMemory — pure function)
  → applyDeltas.js (apply world/arc/objective mutations)
  → narrativeGraph.js (append new Scene node to branching story tree)
  → GameScreen.jsx re-renders new Prose + Paths
```

### LLM Message Structure

Each scene call uses two roles:
- **system**: role declaration + RULES + OUTPUT SCHEMA (static across all scenes)
- **user**: world state + companions + history + player path + task (dynamic per scene)

One call per scene; one automatic retry on JSON parse failure. Token budget: 1600 for non-gpt5, 2400 for gpt-5 models.

### Key Layers

**`src/state/`** — Game state management
- `types.js`: JSDoc type definitions for `GameMemory`, `Companion`, `WorldState`, `Arc` — the canonical data shapes
- `updateFromAIPacket.js`: Pure function merging AI Packet into current Memory; the heart of state transitions
- `applyDeltas.js`: Applies structured mutations to world, arc, and objectives
- `narrativeGraph.js`: Maintains a branching tree of Scene nodes (each player Path creates a branch); exports `buildSceneSegments`, `createNarrativeNode`, `insertNarrativeNode`
- `migrateMemory.js`: Handles save data upgrades; maps old field names (`story`→`prose`, `choices`→`paths`, `currentScene`→`sceneIndex`) to current shape

**`src/utils/`** — LLM orchestration and game logic
- `buildUnifiedPrompt.js` (exports `buildScenePrompt`): Constructs `{ system, user }` for LLM calls; encodes world state, companion memory, scene history
- `storyParser.js` (exports `extractAndNormalizeAiResponse`, `parseGeneratedScene`): Robust JSON extraction with 4-strategy fallback; returns null on failure (triggers retry)
- `AI-chat.jsx` (exports `generateScene`, `generateSceneStream`): Wraps LLM client; `generateScene` is the main call path
- `emotionCalculator.js`: Derives companion emotional states from relationship history
- `phaseOutManager.js`: Companion lifecycle — assigns purpose, manages phaseout countdown (active → phased_out → gone)
- `updateCharacter.js`: Merges incoming character trait updates with stored companion memory
- `memoryManager.js`: Appends Prose/Summary/Path to rolling Memory arrays
- `saveSystem.js`: Multi-slot localStorage save/load (stores timestamp, options, full Memory, graph state)

**`src/services/`** — External communication
- `llmClient.js`: Low-level fetch wrapper for the API proxy; handles timeouts, SSE streaming, and error normalization

**`src/components/`** — React UI
- `App.jsx`: Top-level routing — `MainMenu → StartScreen/LoadGameScreen → GameScreen`
- `GameScreen.jsx`: Central game loop; holds `GameMemory` state, triggers AI calls, owns narrative graph; key state: `segments`, `displayedPaths`, `gameMemory`
- `GameScreenComponents/`: `HeaderBar` (world state HUD), `StoryDisplay` (animated text), `ChoiceGrid` (4 paths with phase transitions: idle → chosen → loading)
- `StartScreenComponents/`: Onboarding steps (`gender-choice`, `genre-choice`, `protagonist-choice`, `setting-choice`, `tone-choice`)
- `NarrativeBranchView.jsx`: Visual explorer of the narrative graph tree (pan/zoom canvas)
- `SceneLog.jsx`, `characterLog.jsx`: Debug panels for transcript and relationship inspection

**`server/server.js`** — Express API proxy
- Forwards requests to OpenAI with auth headers
- Enforces JSON response format (`response_format: { type: 'json_object' }`)
- Dynamic token budget: gpt-5 models get 2400 tokens; others get 1600
- Supports SSE streaming via `stream: true` in request body
- Health check at `GET /api/health`

### LLM Output Schema (Packet)

The LLM always returns strict JSON (no markdown). Key fields:
```json
{
  "title": "string",
  "prose": "string (~200-350 words)",
  "paths": ["string", "string", "string", "string"],
  "characters": [...],
  "summary": "string",
  "sceneTags": [...],
  "objectivesDelta": [...],
  "locationDelta": {...},
  "companionsDelta": [...],
  "arcDelta": { "tension": 0, "beat": 0, "chapter": 0 }
}
```
